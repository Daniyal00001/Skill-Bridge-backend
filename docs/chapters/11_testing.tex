\section{Data Seeding}

The \texttt{prisma/} directory contains a suite of TypeScript seed scripts used to populate the database with realistic test data:

\begin{table}[H]
\centering
\caption{Database Seed Scripts}
\begin{tabularx}{\textwidth}{lX}
\toprule
\textbf{File} & \textbf{Content} \\
\midrule

\texttt{seed\_metadata.ts}  & Populates the skills catalogue, categories, subcategories, languages, and locations. \\
\texttt{seedFreelancers.ts} & Creates 50 realistic freelancer accounts with complete profiles, skills, portfolio items, and token balances. \\
\texttt{seedClients.ts}     & Creates 20 client accounts with company profiles and posted projects in various statuses. \\
\end{tabularx}
\end{table}

Seeds are executed with:

\begin{lstlisting}[language=bash, caption={Running the Database Seed}]
npm run seed
# or
npx tsx prisma/seed.ts
\end{lstlisting}

The seed data provides a fully populated database for development and demonstration purposes, including freelancers with varying SkillToken balances, clients with open projects, proposals in multiple states, and active contracts with milestones.

\section{Frontend Validation Testing}

All frontend pages were manually tested in both light and dark themes across viewport widths of 375px (mobile), 768px (tablet), and 1920px (desktop). The following key flows were validated:

\begin{enumerate}
  \item \textbf{Full registration-to-dashboard flow}: Signup â†’ OTP verification â†’ Role selection â†’ Dashboard.
  \item \textbf{Freelancer onboarding}: All 8 steps of the onboarding wizard, including image uploads and draft saving.
  \item \textbf{Project posting}: All 8 steps of the PostProject form, including draft save and publish.
  \item \textbf{Proposal submission}: Browse projects â†’ View project â†’ Submit proposal â†’ Verify token deducted.
  \item \textbf{Contract workflow}: Accept proposal â†’ View contract â†’ Submit milestone â†’ Approve milestone.
  \item \textbf{AI scoping}: Enter project description â†’ Receive structured output â†’ Pre-fill PostProject form.
  \item \textbf{Admin flows}: Login as admin â†’ Ban user â†’ Resolve dispute â†’ Grant tokens.
\end{enumerate}

\section{TypeScript Type Safety}

TypeScript's static type system provided a continuous layer of correctness checking throughout development. The Prisma-generated client types ensured that all database queries and mutations were type-correct at compile time, preventing entire categories of runtime errors that are common in untyped JavaScript backends. Running \texttt{npx tsc --noEmit} reported zero type errors in the final submission codebase.

\section{Known Limitations}

\begin{itemize}
  \item No automated unit tests or integration tests are implemented (planned future work).
  \item Payment gateway integration is simulated via status fields; no real money flows are tested.
  \item Real-time messaging (Socket.IO) is architecturally planned but not fully implemented in the current version â€” chat rooms and messages are persisted but live push is handled via polling.
  \item OTP rate-limiting is not yet implemented, leaving the OTP endpoint theoretically vulnerable to brute-force within the 10-minute window.
\end{itemize}
